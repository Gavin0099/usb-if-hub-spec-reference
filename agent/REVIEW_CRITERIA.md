# Review Criteria — USB Hub Spec Contract

A change is acceptable only if all checks pass:

1. Every normative claim maps to authority level A source.
2. Every version-specific statement has explicit version scope.
3. Every USB 2.1 statement is expressed as a delta from USB 2.0 unless proven otherwise.
4. Every USB4 statement is marked as topology/router context unless section-level evidence proves equivalence.
5. Compliance documents are not used as sole normative evidence for field definitions.
6. Index/search pages are not used as normative evidence.
7. Community/mirror sources are not used as final authority when official sources exist.
8. Missing section anchors are marked provisional, not verified normative.
9. Repo boundary is preserved: reference clarification, not firmware behavior override.
